# ES-001 — CAP Database Standards
**Version:** 1.0
**Status:** Approved
**Effective:** VS05Z
**Governs:** All platform and future engine database tables

---

## 1. Primary Keys

Every table **must** use a UUID primary key:

```sql
id UUID PRIMARY KEY DEFAULT gen_random_uuid()
```

In Prisma:
```prisma
id String @id @default(dbgenerated("gen_random_uuid()")) @db.Uuid
```

**Prohibited:**
- `SERIAL` / `BIGSERIAL`
- `INT @id @default(autoincrement())`
- Identity columns
- Any integer primary key

---

## 2. Foreign Keys

Every foreign key column **must** reference a UUID:

```prisma
userId    String @db.Uuid
tenantId  String @db.Uuid
entityId  String @db.Uuid
roleId    String @db.Uuid
```

**Never:**
```prisma
userId   Int
tenantId Int
```

---

## 3. Tenant Identifier

All tenant-scoped tables use:

```prisma
tenantId String @db.Uuid
```

This ensures future SaaS deployments and multi-region routing work without integer ID conflicts.

---

## 4. Mandatory Audit Columns

Every mutable platform table **must** contain all of the following:

| Column | Type | Default | Notes |
|--------|------|---------|-------|
| `createdAt` | `DateTime` | `@default(now())` | UTC timestamp |
| `createdBy` | `String? @db.Uuid` | null | Actor UUID |
| `updatedAt` | `DateTime` | `@updatedAt` | Auto-managed |
| `updatedBy` | `String? @db.Uuid` | null | Actor UUID |
| `isDeleted` | `Boolean` | `@default(false)` | Soft delete flag |
| `deletedAt` | `DateTime?` | null | When deleted |
| `deletedBy` | `String? @db.Uuid` | null | Who deleted |
| `version` | `BigInt` | `@default(1)` | Optimistic concurrency |

---

## 5. Soft Delete

All platform tables **must** support logical delete. Physical `DELETE` is never used on platform data.

Soft delete columns: `isDeleted`, `deletedAt`, `deletedBy`

All `findMany` / `findFirst` queries **must** include `isDeleted: false` in where clause (enforced via RuntimeDataEngine).

---

## 6. Optimistic Concurrency

Every mutable table **must** include:

```prisma
version BigInt @default(1)
```

Increment on every update. Compare before write. Throw `ConcurrencyConflictError` on mismatch.

---

## 7. Time Zone

All timestamps **must** be stored in **UTC**.

- Never store local server time
- Never store timezone-adjusted time in the database
- Client displays convert UTC → user local time in the UI layer

---

## 8. Naming Conventions

| Element | Convention | Example |
|---------|-----------|---------|
| Tables | `snake_case` | `entity_records`, `platform_modules` |
| Columns | `snake_case` | `created_by`, `tenant_id` |
| Indexes | `snake_case` | `idx_entity_records_tenant` |
| No quoted identifiers | | Never `"MyTable"` |
| No mixed casing | | Never `createdAt` in SQL |

---

## 9. Runtime Table Inheritance

`SchemaPlatformEngine` **must** generate runtime tables that follow identical standards:

- UUID `id` column
- `created_by UUID`, `updated_by UUID`, `deleted_by UUID`
- `is_deleted BOOLEAN DEFAULT false`
- `deleted_at TIMESTAMPTZ`
- `version BIGINT DEFAULT 1`

Platform tables and runtime tables are governed by the same rules.

---

## 10. Business Fields Exception

The following are **never** converted to UUID:

- Quantities (`qty`, `quantity`, `stock_qty`)
- Amounts (`amount`, `price`, `cost`, `total`)
- Sequence / counter fields (`sequence`, `current_sequence`)
- Year fields (`year`, `fiscal_year`)
- Business record numbers (`DEP-000001`, `EMP-000042`)
  - These use `recordNumber String` and remain separate from `id UUID`

---

## 11. Record Numbers

Business record numbers (e.g., `DEP-000001`) are:

- Stored in a separate `recordNumber String` column
- Generated by `PlatformNumberSeriesEngine`
- Human-readable business identifiers
- Completely separate from the internal `id UUID`
- **Never** used as primary or foreign keys

---

## Compliance

All future CAP engines **must** be compliant with ES-001 before their milestone is marked complete:

- VS06 Runtime Application Engine
- VS07 Workflow Engine
- VS08 Document Engine
- VS09 Notification Engine
- All domain engines (HSE, Catering ERP, Reporting, AI)

Non-compliant tables discovered during review must be corrected in the same milestone that introduces them.
