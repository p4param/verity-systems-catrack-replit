# ADR-008-017 — Workspace Membership Model

**Status:** Accepted

**Engine:** VS08 – License, Subscription & Tenant Management Engine

**Milestone:** VS08A – Tenant Foundation

---

# Context

A TenantMembership establishes organizational affiliation between a Platform User and a Tenant.

However, belonging to a Tenant does not automatically grant access to every operational workspace owned by that Tenant.

Operational access must be granted explicitly.

---

# Decision

Introduce **WorkspaceMembership** as the aggregate representing a User's participation within a specific TenantWorkspace.

A WorkspaceMembership shall always reference an existing TenantMembership.

WorkspaceMembership grants operational access.

TenantMembership grants organizational affiliation.

---

# Architectural Model

```text
PlatformUser
        │
        ▼
TenantMembership
        │
        ▼
WorkspaceMembership
        │
        ▼
TenantWorkspace
```

---

# Responsibilities

WorkspaceMembership is responsible for:

* Workspace association
* TenantMembership association
* Workspace-level role
* Workspace lifecycle

WorkspaceMembership is **not** responsible for:

* Authentication
* Tenant participation
* Licensing
* Billing
* Runtime authorization evaluation

---

# Membership Lifecycle

```text
Invited
        │
        ▼
Active
   ▲    │
   │    ▼
Suspended
        │
        ▼
Removed
```

Removed is terminal.

---

# Membership Rules

* Every WorkspaceMembership must reference an existing TenantMembership.
* WorkspaceMembership cannot exist without TenantMembership.
* A User may belong to multiple Workspaces within the same Tenant.
* A User may hold different roles in different Workspaces.
* Only one active WorkspaceMembership may exist for a given `(Workspace, TenantMembership)` pair.
* Removing a WorkspaceMembership never removes the TenantMembership.

---

# Consequences

This model enables:

* Department-level security
* Regional workspaces
* Multiple operational environments
* Fine-grained workspace access
* Future metadata-driven authorization

---

# Alternatives Considered

### Direct User → Workspace relationship

Rejected.

Would duplicate tenant validation and bypass organizational affiliation.

### Workspace owned User records

Rejected.

Violates identity separation and creates synchronization issues.

---

# Decision Summary

WorkspaceMembership establishes operational participation while TenantMembership remains the organizational boundary.
