# ADR-008-012 — PlatformApplicationPackage Versioning

**Status:** Accepted

**Engine:** VS08 – License, Subscription & Tenant Management Engine

**Milestone:** VS08A – Tenant Foundation

---

# Context

A PlatformApplication represents the identity of a business application.

A PlatformApplicationPackage represents a deployable version of that application.

The platform must support:

* Version history
* Rollback
* Blue/Green deployments
* Canary deployments
* Marketplace distribution
* Controlled upgrades

A consistent versioning strategy is therefore required.

---

# Decision

PlatformApplicationPackage versions shall follow **Semantic Versioning 2.0.0 (SemVer)**.

Examples:

* 1.0.0
* 1.0.1
* 1.1.0
* 2.0.0
* 2.1.0-beta.1
* 2.1.0-rc.1

---

# Rules

## Version Scope

Version uniqueness is enforced **per PlatformApplication**.

Example:

Catering ERP

* 1.0.0
* 1.1.0

HSE

* 1.0.0
* 1.1.0

Both applications may use identical version numbers.

---

## Immutability

After publication:

* Version cannot change.
* Package contents cannot change.
* Manifest cannot change.
* Metadata affecting deployment cannot change.

A published package is immutable.

---

## New Releases

Any functional change requires creation of a new PlatformApplicationPackage.

Existing packages are never modified.

---

## Storage

The version is stored as a string.

Validation occurs within the domain layer.

Persistence must not parse version semantics.

---

## Ordering

Database ordering is not authoritative.

Version comparison and upgrade decisions are performed by platform services using SemVer rules.

---

# Consequences

The platform gains:

* Predictable upgrades
* Safe rollback
* Deterministic deployments
* Marketplace compatibility
* Future support for upgrade policies

---

# Alternatives Considered

### Integer Build Numbers

Rejected.

Do not express compatibility.

---

### Date-Based Versions

Rejected.

Do not express breaking changes.

---

### Custom Version Formats

Rejected.

Increase implementation complexity and reduce interoperability.

---

# Decision Summary

PlatformApplicationPackage versions are immutable, validated using Semantic Versioning 2.0.0, unique within a PlatformApplication, and interpreted by platform services rather than the persistence layer.
