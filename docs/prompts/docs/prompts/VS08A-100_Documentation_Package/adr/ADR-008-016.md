# ADR-008-016 — Tenant Membership Model

**Status:** Accepted

**Engine:** VS08 – License, Subscription & Tenant Management Engine

**Milestone:** VS08A – Tenant Foundation

---

# Context

The CAP Platform requires a clear separation between platform identity and tenant membership.

Users authenticate once through the platform.

They may participate in one or more Tenants.

Tenant membership determines organizational affiliation rather than identity ownership.

---

# Decision

Introduce **TenantMembership** as the aggregate representing the relationship between a Platform User and a Tenant.

A User exists independently of any Tenant.

A TenantMembership links one User to one Tenant.

---

# Architectural Model

```text
Platform User
        │
        │ 1
        ▼
TenantMembership
        │
        │ n
        ▼
WorkspaceMembership (future)
```

---

# Responsibilities

TenantMembership is responsible for:

* Membership identity
* Tenant association
* User association
* Membership lifecycle
* Tenant-level role assignment

TenantMembership is **not** responsible for:

* Authentication
* Workspace permissions
* Licensing
* Billing
* Runtime authorization

---

# Membership Lifecycle

```text
Invited
        │
        ▼
Active
   ▲    │
   │    ▼
Suspended
        │
        ▼
Removed
```

`Removed` is terminal.

---

# Membership Rules

* A User may belong to many Tenants.
* A Tenant may contain many Users.
* Only one active membership may exist per `(Tenant, User)` pair.
* Membership does not create the User.
* Removing a membership never deletes the User.

---

# Consequences

This model enables:

* Multi-tenant users
* External consultants
* Shared support teams
* Enterprise organizations
* Future federation

---

# Alternatives Considered

### User belongs directly to Tenant

Rejected.

Prevents cross-tenant participation.

### Duplicate User per Tenant

Rejected.

Creates identity fragmentation and synchronization issues.

---

# Decision Summary

TenantMembership models organizational participation while Platform User remains the single source of identity.
