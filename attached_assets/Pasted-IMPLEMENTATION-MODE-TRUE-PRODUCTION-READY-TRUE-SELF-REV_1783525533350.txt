Platform Module Permission Mapping

========================================================

OBJECTIVE

Redesign the "Permissions" tab inside the Edit Platform Module dialog.

Remove the existing "Minimum Role Required" dropdown.

Instead, integrate the existing Authorization Engine permission catalog.

The administrator should be able to assign permissions to the Platform Module using the same grouped permission UI already used in Role Management.

DO NOT create another permission system.

Reuse the existing permission tables, repositories, APIs and services.

The Platform Module should only reference permissions.

Roles continue to receive permissions from the Authorization Engine.
