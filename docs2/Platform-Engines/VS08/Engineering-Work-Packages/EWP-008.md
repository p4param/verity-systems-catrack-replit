# EWP-008 — Subscription

**Engine:** VS08 – License, Subscription & Tenant Management Engine

**Milestone:** VS08B – Commercial Foundation

**Capability Contract:** CC-008 – Subscription

**Status:** Approved for Implementation

---

# Objective

Implement the **Subscription** aggregate exactly as defined by the approved CAP architecture (VS08B Commercial Foundation).

Subscription establishes commercial agreements between Tenants and the CAP platform while preserving complete separation between identity, commercial contracts, runtime authorization, and billing execution.

---

# Scope

Implement only the Subscription aggregate.

---

## Domain

Implement:

* Aggregate Root (`Subscription` — mapped to database table `tenant_subscriptions`)
* Commercial Subscription lifecycle state machine (Draft, Trial, Active, Suspended, Expired, Cancelled, Archived)
* Tenant association & SubscriptionPlan association
* Domain validation rules & date invariants
* Provider-neutral `externalReferenceId` field (removed `planCode` and `maxLicenses` per architecture decision)

---

## Persistence

Implement:

* Prisma model (`TenantSubscription` mapped to `tenant_subscriptions` with fields `externalReferenceId`, removing `planCode` and `maxLicenses`)
* Database migration script
* Repository interface (`ISubscriptionRepository`)
* Repository implementation (`SubscriptionRepository`)

## Architecture Notes & Future Triggers

* **Subscription → License Relationship:** Documented in domain model. A `Subscription` produces one or more `License` aggregates (CC-009).
* **Future Commercial Publish Trigger:** Noted in `SubscriptionService`. Lifecycle changes (e.g. activation, renewal, cancellation) will trigger the Commercial Publish Pipeline (CC-014) to regenerate the immutable runtime `CommercialManifest`.

---

## Validation

Validate:

* Tenant exists and is Active.
* SubscriptionPlan exists and is Active.
* Code uniqueness per Tenant (`tenant_id, code`).
* StartDate precedes EndDate.
* Trial status requires future TrialEndDate.
* Immutable associations (`tenant_id`, `subscription_plan_id`).
* Valid state machine lifecycle transitions.

---

## Explicit Exclusions

Do **NOT** implement:

* License generation or authorization enforcement (CC-009 / EWP-009)
* Entitlement resolution engine (CC-010 / EWP-010)
* Feature flag evaluation engine (CC-011 / EWP-011)
* Usage policy resolution engine (CC-012 / EWP-012)
* Commercial Manifest generation (CC-013 / EWP-013)
* Direct payment processing or billing provider webhooks (ADR-008-024)

### Unit Tests

Verify:

* Aggregate creation and reconstitution
* Lifecycle transitions (Draft → Trial → Active → Suspended → Expired → Cancelled → Archived)
* Edge cases: invalid transitions from terminal states (`Archived`), expired trial handling, renewal from expired state, immediate cancellation.
* Immutability of `tenantId` and `subscriptionPlanId`
* Date validation invariants
* Service orchestration

Verify:

* Prisma CRUD operations
